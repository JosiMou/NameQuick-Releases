Release date: 2026-06-07

New

  • Search inside file contents. NameQuick can now find text inside PDFs, images, scanned documents, and supported document and text files, so files can show up even when the matching words are not in the file name.

Improvements

  • Clearer content search progress. Rebuilding, updating, and cleaning the content index now show visible status in Settings and the main window, so you can tell when NameQuick is still checking files.
  • Newer local Gemma 4 models. NameQuick now recommends newer Gemma 4 local models that are designed to use less memory, making local processing a better fit for more Macs.
  • More reliable local model setup. NameQuick is better at choosing and preparing local models for your Mac, including clearer handling for Gemma, MLX, LM Studio, and Ollama workflows.
  • Clearer local Gemma upgrades. NameQuick can now guide you from older local Gemma models to the newer recommended setup without changing your current selection until the replacement is installed.
  • Steadier local model downloads. Local model download cards now keep their progress, size, speed, and status more stable while a model is installing, especially near the end of an Ollama download.
  • Clearer local model labels. Older local model rows now avoid contradictory badges, so it is easier to tell which model is recommended and which one is being replaced.
  • More dependable templates and custom prompts. Saved templates and custom prompts are handled more consistently when files and document inputs are used in naming workflows.
  • Smoother content match browsing. Content matches now stay focused while you review results, with less panel reset behavior and no unnecessary loading loop while search results are already known.
  • More useful rule previews. Rule setup is better at finding sample files from watched folders, so testing a rule should show real examples more reliably.

Bug Fixes

  • Fixed content search rebuilds that appeared to do nothing. Rebuild and clear actions now report progress and completion more reliably instead of silently leaving the index unchanged.
  • Fixed stale content search results after folder changes. Removing a folder, restoring files, or changing files inside a watched folder now updates the content index so old matches are cleaned up and new text can be found.
  • Fixed content indexing during active file writes. NameQuick now waits for changed files to settle before reading their contents, reducing failed or partial search results when another app is still saving.
  • Fixed content search result flicker. Opening and switching content matches should no longer reset the side panel or selected match unexpectedly.
  • Fixed local processing controls that could appear in the wrong place. Local processing settings are now presented more consistently with the current provider setup.
  • Fixed folder move rules. Move destinations such as Processed are now handled as folders, so matching files can be moved into the intended folder instead of being treated like a filename.
  • Fixed saved move rules during file discovery. Saved move rules are now applied before a discovered batch is submitted, so files can land in the expected folder during automatic processing.
