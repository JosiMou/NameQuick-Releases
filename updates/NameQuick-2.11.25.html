Release date: 2026-05-03

New

  • Added an Audit log for rename runs. You can now review recent processing details, understand why a filename needed recovery, and clear the log when you no longer need it.
  • Added clearer field review for templates. Template tests now highlight which extracted details look confident and which ones may need a closer look before you use the result.

Improvements

  • Made custom template filenames more reliable. NameQuick now handles templates that combine fixed fields, sample invoice wording, and custom instructions with fewer formatting surprises.
  • Improved recovery guidance for missing file details. When a rename cannot be completed cleanly, NameQuick gives more specific next steps so you can update the instructions or source file and try again.
  • Made app diagnostics more privacy-conscious. Diagnostic context is now grouped into safer, broader categories so troubleshooting can improve without exposing unnecessary file details.

Bug Fixes

  • Fixed a stale subscription loading state. The license screen is less likely to keep showing a loading state after subscription information has already been resolved.
  • Fixed local AI output handling edge cases. Local providers now recover more cleanly when structured responses come back in a slightly different shape than expected.
  • Fixed hybrid template assembly issues. Template-based filenames now better preserve the intended field order, visible labels, and file extension.
