Release date: 2026-05-04

New

  • Added shareable processing reports. You can now copy a clear report from a processing run, making it easier to review what happened or share details when asking for help.
  • Added clearer processing issue cards. NameQuick now groups processing problems into easier-to-scan cards with clearer labels and next steps.

Improvements

  • Made local AI limits easier to manage. Advanced budget controls for Ollama, LM Studio, and MLX are now tucked behind a cleaner disclosure section while keeping your saved settings intact.
  • Improved local provider guardrails. NameQuick now checks local model limits more consistently before sending work, helping prevent oversized requests and keeping provider behavior predictable.
  • Kept selected AI providers more consistent. BYOK and local provider flows now preserve the provider you chose more reliably during file preparation and processing.
  • Improved processing history details. The audit trail now records richer outcome details so completed runs are easier to understand after the fact.

Bug Fixes

  • Fixed missing details in processing feedback. Issue reports now preserve missing-field information more reliably instead of dropping useful context.
  • Fixed local model budget settings appearing too prominently. Detailed limits now stay available without crowding the main local AI settings view.
  • Fixed processing feedback surfacing in the wrong place. NameQuick now routes run results, HUD messages, and history details through a more consistent presentation flow.
