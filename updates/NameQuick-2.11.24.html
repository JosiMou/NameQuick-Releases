Release date: 2026-05-02

New

  • Added stronger local AI setup and loading. NameQuick can now install and load local models through more of the provider’s own app experience, including LM Studio, with clearer progress while setup is still running.
  • Added recovery guidance when file details are missing. When a rename request needs information that is not visible in the file, NameQuick can now explain what is missing and suggest a practical next step instead of returning a confusing filename.

Improvements

  • Made local provider setup more reliable. Local AI readiness checks now better distinguish app availability, endpoint health, model status, and runtime state so NameQuick can choose the right recovery path.
  • Improved local model downloads. LM Studio downloads now show clearer progress and avoid switching to a local command-line fallback when you are using a remote LM Studio server.
  • Made custom naming prompts more predictable. Filename results now keep the original file extension automatically, preserve visible names such as companies and people, and handle non-English descriptive words without rejecting proper nouns.
  • Improved managed Gemini validation. NameQuick now keeps provider routing clearer for managed Gemini testing and better validates request shaping before traffic reaches the backend.
  • Reduced interface noise during processing. The processing trail is now kept out of the main experience unless debug tools are enabled.

Bug Fixes

  • Fixed local provider readiness false starts. NameQuick is less likely to report a local provider as ready before the selected endpoint and model are actually usable.
  • Fixed recovery messages for optional template fields. Optional defaulted fields are now handled more cleanly so missing-data guidance focuses on the details that truly need attention.
  • Fixed filename prompt failures around extensions and language. Prompt examples that mention .pdf or another extension are now treated as formatting intent, while missing-content errors consistently use the same clear failure format.
