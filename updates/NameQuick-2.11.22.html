Release date: 2026-04-26

New

  • Refreshed the installer artwork. The drag-to-Applications window now uses updated release packaging art.

Improvements

  • Made MLX local model readiness easier to control. NameQuick now has clearer Start, Stop, and Keep Ready behavior, and avoids starting the runtime just because you view settings.
  • Made custom template and freeform filenames more reliable. Local filename prompts now use a safer flow that extracts file facts first, then builds filenames from those facts.
  • Improved Ollama PDF processing. Difficult PDFs can finish the rename first while optional metadata is handled separately when needed.
  • Made structured presets more consistent. Template previews, transformations, and final filenames now follow the same model-authored filename path.

Bug Fixes

  • Fixed malformed AI output being accepted as filenames. NameQuick now rejects placeholder text, prompt instructions, XML wrappers, and malformed filename responses more reliably.
  • Fixed stale MLX readiness state. NameQuick no longer reuses a local runtime that has already stopped or moved.
  • Fixed Ollama model recovery and selection edge cases. Missing, unsupported, or unavailable local models are shown and handled more clearly.
  • Fixed BYOK trial login and account switching edge cases. NameQuick preserves the right access and recovery state more reliably.
  • Fixed provider routing safeguards. BYOK and non-managed setups now fail clearly instead of silently moving to a managed Gemini path.
