Release date: 2026-07-02

This update makes Quick Process corrections more reusable, improves local model recovery, and fixes several cases where rename and setup flows could show stale or confusing state.

New

  • Turn a Quick Process correction into a preset. When you fix a proposed name, NameQuick can suggest a naming recipe for the rest of the batch, apply it, and save it as a preset for future files.
  • Sample files in the first-run library. Fresh installs now show a small demo when there are no files yet, so you can see how rename previews work before adding your own batch.

Improvements

  • Clearer Quick Process results. The drawer now shows before-and-after name changes, a success line when a batch finishes, and an Undo all action for finished Quick Process renames.
  • Smarter default Smart Rename style. The default naming prompt now adapts better to different file types, producing more consistent names before you make custom templates.
  • Better local model setup and switching. MLX, LM Studio, and other local provider screens now explain readiness, model loading, and repair paths more clearly before a run starts.
  • More helpful blocked-queue recovery. Stalled processing queues now show direct recovery actions or a dismiss option instead of leaving jobs sitting without a clear next step.

Bug Fixes

  • Fixed Managed AI showing the wrong setup warnings. Managed mode no longer asks for local-model or API-key fixes that do not apply to Managed runs.
  • Fixed broken MLX runtime installs leading to a dead end. NameQuick now offers a Repair action when the local MLX runtime needs to be rebuilt.
  • Fixed provider setup getting stuck during onboarding. Setup now offers a recovery path when saved provider checks need attention before templates or custom prompts can run.
  • Fixed rename previews flickering during batch runs. Proposed names should no longer replay or flip between old and new values while a batch settles.
  • Fixed Quick Process drawer bleed-through. The floating drawer now keeps background controls from showing through while it is open.
  • Fixed BYOK trial processing loops. Queued work no longer repeatedly pauses and resumes while Self-Managed trial access is being checked.
