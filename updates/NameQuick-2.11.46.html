Release date: 2026-06-21

This update makes review-first workflows safer, improves rules history and undo, strengthens watch-folder importing, and makes local AI, templates, and licensing more reliable.

New

  • Review hold for automatic renames. A new setting can send watched-folder and automatic rename suggestions to Review before files are changed, so hands-off workflows can still have a check step.
  • Move-back actions for rule-organized files. Recent Renames now shows files moved, trashed, or archived by rules, with Move back or Put back actions where reversal is supported.

Improvements

  • Safer first-run renaming. Fresh installs now start in Review mode, so new users see proposed filenames before NameQuick writes changes to disk.
  • More predictable watched-folder setup. NameQuick handles existing files during watch-folder setup more reliably, with clearer baselines for future arrivals.
  • Faster large-folder importing. NameQuick scans and imports big folders with less memory use and less window freezing, especially when folders contain many existing files.
  • More dependable templates and custom prompts. Imported templates keep their names, rules, and test behavior more consistently, and older saved templates recover more predictably.
  • Stronger local AI readiness. Local MLX setup now reports incompatible or failed model states more clearly and can guide users toward repair instead of silently failing.
  • Clearer licensing and self-managed setup recovery. Checkout and upgrade flows now show clearer messages when a completed purchase cannot be matched to the right account.
  • Smoother selection and search feedback. File selections, footer counts, action buttons, and content-search status stay in sync more reliably while files and indexes update.

Bug Fixes

  • Fixed repeat-rename undo. Undo now returns a file to the immediate previous name instead of jumping back to an older name or inventing a duplicate “restored” filename.
  • Fixed missing history for template and custom-prompt test renames. Files renamed from those authoring flows now appear in Recent Renames and can be undone.
  • Fixed dotted identifiers in filenames. Case numbers, versions, and other values with interior dots should no longer be truncated or gain duplicate extensions.
  • Fixed rule moves and trash actions leaving stale library rows. Moving back or putting back a rule-organized file now keeps the library row aligned with the real file.
  • Fixed silent empty folder adds. When adding existing files finds no supported files, NameQuick now tells you instead of appearing to do nothing.
  • Fixed duplicate watch-folder backfill processing. Existing files in watched folders should no longer be queued twice or skipped incorrectly when catalog rows need repair.
  • Fixed Quick Process history gaps. Quick Process renames are recorded more reliably before completion, improving recovery if the app closes at the wrong moment.
  • Fixed Finder metadata setting confusion. Turning on AI-generated file descriptions now enables the required Finder metadata setting instead of looking unavailable.
