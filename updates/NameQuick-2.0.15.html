NameQuick v2.0.15

This release makes Templates rock‑solid with Claude, fixes an OpenAI schema edge case, and keeps spreadsheets fast and stable.

Highlights

  • Templates (Claude): Reliable structured output with consistent results.
  • OpenAI schema: Fixes rare “type: None” error in structured output.
  • Spreadsheets: Stable memory + faster previews for large XLSX files.
  • Error messages: Clearer, provider‑specific guidance.

Improvements

  • Templates (Claude)
    • Uses Claude Tools with an input schema to return clean JSON (no chatty responses).
    • Normalizes template field keys under the hood for tool compatibility; your template labels are unchanged.
  • Spreadsheet handling
    • Size‑aware parsing and smarter sampling keep memory bounded on large workbooks.
    • Faster, steadier previews for DOCX/PPTX/XLSX.
  • Error messages
    • Clearer document guidance and provider‑specific mapping (no generic attributions).
    • Unified wording for unsupported actions; suggests the next best path.

Fixes

  • Templates (Claude): Resolves “response_format: Extra inputs are not permitted” by switching to Tools.
  • OpenAI: Fixes rare structured‑output error (“schema must be type: object, got None”).
  • Office previews: Prevents spikes from embedded thumbnails; steadier memory.
  • Spreadsheet parsing: Resolves a leak in XLSX handling and aborts cleanly on extraction failures with clear guidance.

Thanks for using NameQuick and for all the feedback — it helps a ton.
