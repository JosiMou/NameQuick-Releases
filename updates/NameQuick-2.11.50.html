Release date: 2026-06-25

This update adds camera metadata fields for photo naming, improves local MLX model setup, cleans up plan management in Settings, and fixes several processing-status issues.

New

  • Camera metadata for templates, custom prompts, and rules. You can now use Date Taken, Camera Make, and Camera Model from image metadata when building photo-focused naming workflows.

Improvements

  • Faster, clearer MLX model installs. MLX model downloads now show more accurate progress, speed, and ETA, download large model files in parallel, and check available disk space before starting.
  • MLX models are ready sooner after install. When you install the selected MLX model from Settings, NameQuick can warm it in the background so the first rename does not have to wait as long.
  • Clearer plan management in Settings. About now shows fewer irrelevant license controls, uses clearer upgrade wording, and offers a Self-Managed choice from the managed plan flow when that switch is available.
  • Safer custom prompt run controls. Custom prompts stay review-first until the Skip Review option has been earned, keeping automation controls out of the way until they are relevant.

Bug Fixes

  • Fixed stuck processing indicators. The action bar, processing footer, and menu bar spinner should no longer stay busy after a cancelled or completed job has already stopped.
  • Fixed manual processing mode mismatches in watched folders. Manual runs now use the mode shown in the toolbar instead of silently switching to a folder’s automatic watch-folder mode.
  • Fixed disabled run buttons for valid selections. NameQuick now keeps folder membership counts in sync so the Play button stays available when selected files are actually in scope.
  • Fixed Change Plan in Settings. Upgrade and plan-change sheets now present correctly from the Settings window instead of doing nothing.
  • Fixed MLX download stalls. Large MLX model downloads should no longer appear frozen partway through a file, and connection stalls fail with a clearer retry path.
