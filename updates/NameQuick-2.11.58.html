Release date: 2026-07-06

This update refreshes AI setup, AI settings, local model recovery, and inspector behavior, adds optional beta updates, and improves reliability around large batches, watched folders, provider setup, and duplicate filenames.

New

  • Optional beta updates. You can now opt into beta versions from update settings when you want earlier access to upcoming fixes and features.

Improvements

  • Simpler AI setup. First-run AI setup now walks through one decision at a time, keeps local model setup clearer, and makes background downloads easier to follow.
  • Clearer local and cloud AI choices. NameQuick now explains local and cloud AI trade-offs more plainly and opens the right provider setup screen when setup needs attention.
  • Clearer AI settings. The AI settings page now shows your current setup, model choices, and advanced options in one calmer layout, with more accurate OpenRouter key checks.
  • Better local model recovery. LM Studio, MLX, and Ollama setup now show clearer recovery cards, waiting states, and next actions when a runtime or model needs attention.
  • Better watched-folder controls. Watched folders now show Off, Smart Rename, and Preset states more clearly, with recent activity easier to scan.
  • Faster large rename batches. Cloud rename batches can now move through large sets more smoothly and slow down automatically when a provider is busy or rate limited.
  • More flexible duplicate filenames. You can choose how NameQuick numbers duplicate filenames, so conflicts can use the style that fits your workflow.
  • Clearer custom prompt review. Custom prompts now show a more consistent read-back of the naming shape they will produce, making updates easier to review.
  • More comfortable inspector layout. Inspector panels now behave better at smaller window sizes, with route titles staying readable and fewer layout conflicts while resizing.

Bug Fixes

  • Fixed inspector close and toggle issues. Closing review details and using inspector toolbar controls should now work on the first click without leaving the panel or file list in a stuck layout.
  • Fixed partial review forms missing usable details. Review now carries over found values and shows inputs for every empty required field, so incomplete suggestions are easier to finish before applying.
  • Fixed warnings being easy to miss. User-initiated warnings now route to the in-app HUD so they are more visible while you are working.
  • Fixed AI model choices getting out of sync. The model shown in AI settings should now match the model NameQuick will actually use.
  • Fixed Ollama model switching. Switching away from or stopping an Ollama model should now update cleanly even if the previous model was already stopped or missing.
  • Fixed local model status flicker during downloads. Local AI setup should no longer jump between conflicting states while a model is downloading.
  • Fixed OpenRouter setup confusion. NameQuick now validates OpenRouter keys more directly and removes inactive key actions that could make setup feel stuck.
  • Fixed BYOK activation recovery. Devices that were already activated should now be recognized more reliably instead of using an extra device slot after a setup hiccup.
  • Fixed vague provider errors. BYOK provider failures now use more of the provider’s error details, so recovery steps should be clearer.
