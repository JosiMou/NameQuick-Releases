Release date: 2026-07-03

This update adds a safer way to recover from stuck processing, makes queued renames and watch folders more reliable, and improves billing and provider guidance.

New

  • Stop and clear a stuck queue. A new Stop and Clear Queue action lets you stop processing and clear queued work while keeping your files in your library and on disk.

Improvements

  • Clearer Managed trial unlocks. When a trial has ended and unlocking Managed AI will charge now, NameQuick now confirms the charge before continuing.
  • Clearer preset scope. The processing menu now explains when a preset applies to files you rename yourself and when watched folders use their own auto-rename preset.
  • Cleaner Quick Process review rows. If a proposed name already matches the current file name, NameQuick now marks it as already named and hides unnecessary apply actions.
  • Clearer recovery for failed renames. Failed rename rows now keep a more precise reason and next step, so NameQuick can point you to review, AI settings, or local model recovery when that matches the problem.
  • More reliable review after failed runs. Names that need review should stay connected to the right file evidence after the app restores a failed or interrupted run.

Bug Fixes

  • Fixed automatic renames running again after a manual rename. Watch-folder work now rechecks the current file state before renaming, so files already handled manually should not be renamed a second time by queued automatic work.
  • Fixed queue state getting out of sync. Queued work now settles more consistently after relaunch, completion, cancellation, or clearing, so files should not reappear or leave processing controls blocked.
  • Fixed watch folders unexpectedly disappearing. NameQuick now keeps a last-good folder list and can restore it if saved watch folders are accidentally emptied, without bringing back folders you deliberately removed.
  • Fixed unreadable watch-folder settings being overwritten. If NameQuick cannot read the saved folder list, it now preserves the existing data and stops risky saves instead of replacing the folder list.
  • Fixed Managed AI being blocked by local-model setup. Managed runs no longer show local-model recovery prompts or disable Process just because a local provider is unavailable or still loading.
